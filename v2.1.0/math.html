Dynamical systems are systems which evolve over time. In general, such a system has a set of inputs and outputs, both of which may change over time. In contrast, for a static system its output \(\vec{y}\left(t\right)\) only depends on the value of the input \(\vec{u}\left(t\right)\) at that exact point in time:

\[\vec{y}\left(t\right) = \vec{h}\left(\vec{u}\left(t\right)\right)\]

For a dynamic system, the output at a given point in time may also depend on the inputs at other points in time. In the real world, all systems are causal, i.e. the effect always comes after the cause. Thus, for a causal dynamic system, the output at a point in time \(t\) may only depend on the inputs at times before \(t\).

That means that dynamic systems must have some kind of memory to represent the effects of earlier inputs. We say that a dynamical system has a state and we represent that state as a vector. For most practical applications that state has finite dimension.

Continuous-time Systems

MoDyPy supports modelling continuous-time systems in state-space formulation:

\[\begin{split}\vec{x}\left(t_0\right) &= \vec{x}_0 \\ \frac{d}{dt} \vec{x}\left(t\right) &= \vec{g}\left(t, \vec{x}\left(t\right), \vec{u}\left(t\right)\right) \\ \vec{y}\left(t\right) &= \vec{h}\left(t, \vec{x}\left(t\right), \vec{u}\left(t\right)\right)\end{split}\]

Here,

  • \(\vec{x}\left(t\right)\) is a vector that represents the state of

    the system at time \(t\), and

  • \(\vec{x}_0\) is the initial state or initial condition of the system

    at time \(t_0\).

The function \(\vec{g}\) describes the derivative of the state with respect to time. That is, it describes how the state evolves with time. Again, it may also depend on the time, the state and the input vector.

Finding the value of \(\vec{x}\left(t\right)\) for some value of \(t\) based on the equations given above is called an initial value problem, and closed solutions are only known for specific forms of \(\vec{g}\), such as those that are linear. For most other cases, numerical approximations of the solutions are available and also sufficient.

Discrete-Time Systems

While continuous-time systems allow us to easily describe typical physical processes, there are some applications which require us to describe the progression of a system in discrete time steps. In that case, the state and output only change at specific points in time. Usually, these points in time are regularly-spaced, and the time between two consecutive of such points in time is called the period of the system.

A typical formulation you will find is the following:

\[\begin{split}\vec{x}_{k+1} &= \vec{g}\left(k, \vec{x}_k, \vec{u}_k\right) \\ \vec{y}_k &= \vec{h}\left(k, \vec{x}_k, \vec{u}_k\right)\end{split}\]

This kind of system can be easily simulated by a computer, as the state at the respective next time step is explicitly given, while in the continuous case, an initial value problem with differential equations needs to be solved first. This is also why digital controllers or filters implemented in software are typically modelled as discrete-time systems.

System Properties

Systems may be classified according to a set of attributes, such as time-dependency, linearity or autonomy. In the following, a few important attributes for systems are given:

time-invariance

A system is said to be time-invariant if both \(\vec{g}\) and \(\vec{h}\) are independent of the parameter \(t\). A system that is not time-invariant is said to be time-varying.

linearity

A system is said to be linear if both \(\vec{g}\) and \(\vec{h}\) are linear in their parameters \(t\), \(\vec{x}\) and \(\vec{u}\). In that case, both functions can take a specific form and closed solutions are available. For more information, see Linear Time-Invariant Systems. Systems that are not linear are called non-linear.

autonomy

A system is said to be autonomous if both \(\vec{g}\) and \(\vec{h}\) are independent of the input \(\vec{u}\).

Linear Time-Invariant Systems

A very interesting special class of systems are linear and time-invariant systems, or LTI for short. For continuous-time systems, these can be expressed in the following form:

\[\begin{split}\frac{d}{dt} \vec{x} &= A \vec{x} + B \vec{u} \\ \vec{y} &= C \vec{x} + D \vec{u}\end{split}\]

with matrices \(A\), \(B\), \(C\), \(D\). Linear discrete-time systems can be expressed in a similar form:

\[\begin{split}\vec{x}_{k+1} &= A \vec{x}_k + B \vec{u}_k \\ \vec{y}_k &= C \vec{x}_k + D \vec{u}_k\end{split}\]

These systems are particularly interesting, because there are generic methods for solving the differential and difference equations for the state in closed form, for analysing the controllability, observability and stability of such a system and for designing controllers and so-called observers.

However, MoDyPy is able to handle non-linear and linear systems of both the discrete- and the continuous-time variant. Specifically, MoDyPy is designed to handle mixed systems consisting of continuous-time and discrete-time parts, also with mixed periods. It can also derive linear approximations of the dynamics of a continuous-time non-linear system.
